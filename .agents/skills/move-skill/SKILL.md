# Move Skill（スキル移動スキル）

このスキルは、指定されたスキルフォルダを `skills/` ディレクトリへ移動します。
スキルのディレクトリ構造を整理したいときに活用してください。

## 前提条件

- 移動先の `skills/` ディレクトリは、エージェントスキルのリポジトリルート（`agent-skills/`）直下にあることを想定します
- 移動対象はスキルフォルダ（`SKILL.md` を含むディレクトリ）です

## 手順

### 1. 情報の確認

ユーザーのリクエストから以下を確認してください：

- **移動元パス**: 移動したいスキルのフォルダパス（例: `/path/to/my-skill`）
- **移動先ディレクトリ**: `skills/` ディレクトリを使用します（デフォルト: `agent-skills/skills/`）

どちらかが不明な場合は、ユーザーに確認してください。

### 2. SKILL.md の存在確認

移動前に、指定フォルダに `SKILL.md` が存在するか確認してください：

```bash
ls <移動元パス>/SKILL.md
```

`SKILL.md` が存在しない場合は、ユーザーに「指定されたフォルダにはSKILL.mdが見つかりませんでしたが、続行しますか？」と確認を取ってください。

### 3. 移動先に同名フォルダが存在しないか確認

```bash
ls <移動先ディレクトリ>/<スキル名>
```

既に存在する場合は、ユーザーに上書きするか確認してください。

### 4. スキルを移動する

```bash
mv <移動元パス> <移動先ディレクトリ>/
```

**例:**
```bash
mv /Users/snd/GitRepo/my-custom-skill /Users/snd/GitRepo/agent-skills/skills/
```

### 5. 移動完了を確認・報告

移動後に移動先を確認し、ユーザーに結果を報告してください：

```bash
ls <移動先ディレクトリ>/<スキル名>
```

報告内容の例：
> ✅ `my-skill` を `skills/my-skill` へ移動しました。

## 注意事項

- 移動元のパスを間違えないよう、ユーザーと確認してから実行してください
- 移動元と移動先が同じパスにならないよう注意してください
- 読み取り専用のパス（インストール済みスキルの `.agents/skills/` 配下など）からの移動は失敗する可能性があります。その場合は `cp -r` でコピー後、元フォルダの削除をユーザーに確認してください
- 移動後、必要であればリポジトリの `skills-lock.json` の更新についてユーザーに案内してください

## 出力例

```
移動元: ~/Documents/new-skill
移動先: ~/GitRepo/agent-skills/skills/new-skill
✅ 移動が完了しました。
```
