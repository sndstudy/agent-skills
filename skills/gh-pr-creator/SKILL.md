# gh-pr-creator（GitHub PR 作成スキル）

`gh` コマンドを使い、現在のブランチの変更内容からPull Requestを自動作成するスキルです。
差分を分析してPRタイトルとボディを自動生成するため、一貫性のある高品質なPRを素早く作れます。

---

## 手順

### Step 1: 前提確認

まず `gh` コマンドが使えるか、そして認証済みかを確認します。

```bash
gh auth status
```

エラーが出た場合は「`gh auth login` で認証してください」とユーザーに案内して終了してください。

---

### Step 2: 現在のブランチを確認

```bash
git branch --show-current
```

`main` や `master` などのデフォルトブランチに直接いる場合は、「デフォルトブランチからは直接PRを作成できません。フィーチャーブランチに切り替えてください」と伝えて終了してください。

---

### Step 3: リモートへのpush状態を確認

現在のブランチがリモートにpush済みかどうかを確認します。

```bash
git status -sb
```

出力の1行目（`## <branch>...origin/<branch>`）を見てください：

- **追跡ブランチが存在しない**（`origin/<branch>` の表記がない）場合 → リモートにpushされていません
- **`[ahead N]` のみ** → ローカルのコミットがリモートより進んでいます（未push分あり）
- **`[behind N]` または `[ahead N, behind N]`** → リモートとの乖離があります

**pushされていない・未push分がある場合**は、以下を実行してユーザーに確認します：

> 「現在のブランチはリモートにpushされていません（または未pushのコミットがあります）。pushしてからPRを作成しますか？」

ユーザーが同意したら：

```bash
git push -u origin <現在のブランチ名>
```

push失敗時はエラー内容をユーザーに伝えて終了してください。

---

### Step 4: 派生元ブランチを特定する

PR の比較対象となるベースブランチを特定します。

```bash
gh repo view --json defaultBranchRef --jq '.defaultBranchRef.name'
```

これでリポジトリのデフォルトブランチ（通常は `main` または `master`）を取得します。

次に、現在のブランチがそのデフォルトブランチから分岐したか確認します：

```bash
git merge-base --fork-point <デフォルトブランチ名> HEAD
```

分岐点が見つからない場合や、より適切なベースブランチがある場合は、ユーザーに確認してください。

---

### Step 5: 差分を分析する

ベースブランチとの差分を収集します。

**変更ファイル一覧：**
```bash
git diff <ベースブランチ名>...HEAD --name-status
```

**コミット履歴：**
```bash
git log <ベースブランチ名>...HEAD --oneline --no-merges
```

**差分の内容（大きすぎる場合は省略可）：**
```bash
git diff <ベースブランチ名>...HEAD --stat
```

コード変更の詳細が必要な場合（PRの説明を充実させたいとき）：
```bash
git diff <ベースブランチ名>...HEAD
```

差分が非常に大きい（500行以上）場合は `--stat` の結果と変更ファイルの概要のみを使い、全差分の取得はスキップして構いません。

---

### Step 6: PRタイトルとボディを生成する

差分の分析結果をもとに、PRタイトルとボディを日本語で作成しします（ユーザーが英語を希望している場合は英語で）。

#### タイトルの指針
- 変更の「目的」を一言で表現する
- 50文字以内を目安に簡潔に
- プレフィックス（`feat:`, `fix:`, `chore:` など）がコミットに使われていれば踏襲する

**タイトルの例：**
```
feat: ユーザー認証にJWT対応を追加
fix: ログイン時のセッション切れバグを修正
chore: 依存パッケージを最新バージョンに更新
```

#### ボディのテンプレート

```markdown
## 概要
<!-- この PR で何をしたかを簡潔に説明 -->

## 変更内容
<!-- 変更したファイルや機能を箇条書きで -->

## 背景・目的
<!-- なぜこの変更が必要だったか -->

## 確認事項
<!-- レビュワーに確認してほしい点があれば -->
```

差分の内容に応じて各セクションを埋めてください。「確認事項」は特になければ省略して構いません。

---

### Step 7: ユーザーに確認する

PRを作成する前に、生成したタイトルとボディをユーザーに提示して確認を取ります。

```
以下の内容でPRを作成します。よろしいですか？

【タイトル】
<生成したタイトル>

【ベースブランチ】
<ベースブランチ名> ← <現在のブランチ名>

【ボディ】
<生成したボディ>
```

ユーザーが修正を希望した場合は内容を調整し、再確認してから次のステップへ進んでください。

---

### Step 8: PR を作成する

ユーザーの確認が得られたら `gh` コマンドでPRを作成します。

```bash
gh pr create \
  --base <ベースブランチ名> \
  --head <現在のブランチ名> \
  --title "<タイトル>" \
  --body "<ボディ>"
```

**ドラフトPRにしたい場合**（ユーザーが希望したとき）：
```bash
gh pr create \
  --base <ベースブランチ名> \
  --head <現在のブランチ名> \
  --title "<タイトル>" \
  --body "<ボディ>" \
  --draft
```

---

### Step 9: 完了を報告する

PR作成後、`gh pr create` の出力に含まれるPRのURLをユーザーに伝えます。

```
✅ PRを作成しました！
🔗 <PR の URL>
```

---

## 注意事項

- `gh` コマンドは [GitHub CLI](https://cli.github.com/) のインストールが必要です
- `gh auth login` で事前に認証が必要です
- ボディに特殊文字（`"` や改行）が含まれる場合、シェルのエスケープに注意してください。必要であれば一時ファイルに書き出して `--body-file` オプションを使ってください：

```bash
# ボディをファイルに書き出してから使用する場合
cat > /tmp/pr_body.md << 'EOF'
<ボディの内容>
EOF

gh pr create \
  --base <ベースブランチ名> \
  --title "<タイトル>" \
  --body-file /tmp/pr_body.md
```
